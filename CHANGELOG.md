# Changelog

All notable changes to Elektra Skills will be documented in this file.

## [0.5.1] - 2026-03-28

### Added
- You can now find every registry where Elektra Skills is listed or pending in the [Registry Distribution Guide](docs/REGISTRY-DISTRIBUTION.md) -- 14 registries across 3 tiers
- All 6 skills now declare agent compatibility via the `compatibility` frontmatter field, per the [agentskills.io spec](https://agentskills.io/specification)
- GitHub topics set for discoverability: agent-skills, claude-code, cursor, gemini-cli, codex, ai-governance, ai-agent, skills-sh, responsible-ai, windsurf

## [0.5.0] - 2026-03-28

### Added
- Solutions Architect persona via CLAUDE.md with adaptive seniority bands (S1-S4) and 6 auto-detected modes
- 5 governance shell hooks: session-init, cycle-guard, quality-gate, rai-check, token-cap-guard
- Standing Orders dispatch: SO1 Godspeed (14-phase execution), SO2 Project Management, SO3 Quality Delivery
- Responsible AI skill with 7-pillar governance framework and kill-switch protocol
- Autoresearch skill with 5 modes: iterate, security, ship, debug, fix
- Godspeed and godspeed-resume skills for direct 14-phase execution
- Project management skill for GitHub-powered PM routine
- Design review gates at P3.5 (pre-execution) and P4.5 (post-execution)
- Companion skill ecosystem: gstack, superpowers, ui-ux-pro-max, claude-mem, everything-claude-code
- Memory protocol with typed memory files (user, feedback, project, reference)
- Conductor multi-agent orchestration support via claude-peers MCP
- Marketing site at `site/` — standalone Next.js 16 app with Kinetic Silence dark theme
- 6-scene scroll narrative: hero, what-is-elektra, problem, skills-showcase, proof, CTA
- 4-step journey visualization (Install, Configure, Execute, Ship)
- Agent compatibility grid with 8 named AI coding tools
- First session protocol with batched onboarding and auto-discovery
